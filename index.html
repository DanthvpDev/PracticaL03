¿Por qué es importante utilizar etiquetas semánticas para mejorar el SEO y la Accesibilidad?

Daniel Hernandez

¿Qué es SEO?

El SEO (Search Engine Optimization) es es el proceso de optimizar un sitio web y su contenido para mejorar su visibilidad y posición en los resultados de búsqueda orgánicos de los motores de búsqueda. En pocas palabras, es el proceso por el cual el buscador posiciona los resultados de una búsqueda que se generan de forma natural (sin necesidad de invertir en publicidad) "puntuando" la optimización de los elementos dentro de un sitio web (etiquetas HTML, URL amigable, optimización de imágenes y videos, etc.), la optimización de la versión móvil del sitio, la arquitectura del sitio, entre otros. Entre más aspectos del SEO se abarquen, el buscador premiará con una mejor posición al sitio o página web.

En vista de que el buscador organiza los resultados basándose en cómo se estructura el HTML del sitio, es importante que, como desarrolladores, sigamos las mejores prácticas de SEO. De esta manera los sitios de internet que desarrollemos estarán mejor posicionados.

¿Qué es el HTML semántico?

El HTML (HyperText Markup Language) es el lenguaje de etiquetado con el que se estructuran las páginas web. Este lenguaje consta de etiquetas conocidas como "tags" que especifícan el fin o qué función cumple un elemento de la página web. El buen uso de estas etiquetas mejorará el posicionamiento de nuestro sitio o página web. Para ello se utilizan etiquetas semánticas.

¿Qué son las etiquetas semánticas y cuál es su función?

Como se mencionó anteriormente, HTML es un lenguaje de marcado que, en sus inicios, contaba con un número limitado de etiquetas para estructurar texto, divisores, hipervínculos, etc. Sin embargo, muchas de estas etiquetas eran algo ambiguas. Por ejemplo, para crear un menú de navegación, se utilizaba la etiqueta <div>, que también servía para dividir secciones de la página, delimitar pequeños bloques de contenido como artículos, o agrupar información relacionada. Esto hizo que <div> se convirtiera en una etiqueta demasiado genérica. ¿Se estaba utilizando para algo importante o secundario? Esta ambigüedad dificultaba la interpretación tanto para los motores de búsqueda como para los desarrolladores, ya que no indicaba claramente el propósito de cada bloque de contenido.

Con la llegada de nuevas versiones de HTML, se introdujeron etiquetas como <nav>, <section>, y <article>, entre otras. Estas etiquetas tienen un propósito específico: mejorar la legibilidad del código y facilitar la comprensión del contenido tanto para los desarrolladores como para los motores de búsqueda. Estas etiquetas se conocen como etiquetas semánticas. El HTML semántico ayuda a que los motores de búsqueda no solo se concentren en palabras clave sueltas, sino también en el significado y el contexto de las palabras utilizadas. Esto les permite ofrecer resultados más precisos, incluso si el usuario no ingresa exactamente las mismas palabras que aparecen en la página. Además, los motores de búsqueda pueden priorizar ciertos contenidos según las etiquetas utilizadas, dando mayor relevancia a secciones envolventes como <article> o <header>.
